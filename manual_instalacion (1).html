Sección 1

1. Manual de instalación

Este manual describe los pasos generales para preparar el sistema en un entorno local de desarrollo o validación.

Usa el menú lateral para cambiar entre secciones. Cada opción se mostrará como una vista independiente dentro del mismo documento.
Sección 1.1

1.1 Requisitos previos

Antes de instalar el sistema se requiere:

  • Sistema operativo compatible con Node.js y Python.
  • Node.js 20.x.
  • Python 3.
  • Acceso al repositorio del proyecto.
  • Variables de entorno del frontend y backend.
  • Cuenta y proyecto Supabase configurado para producción temprana.
  • Clave de OpenAI configurada para transcripción e interpretación.
  • Arduino IDE para cargar el firmware al ESP32.
  • ESP32 conectado por USB para pruebas de hardware.
Sección 1.2

1.2 Estructura del proyecto

El proyecto se organiza en repositorios y carpetas principales:

/home/abraham/proy_ia_security
|-- frontend/
|-- backend/
|-- firmware/
|-- audios_recibidos/
|-- README.md

La carpeta frontend/ contiene la aplicación web. La carpeta backend/ contiene la API FastAPI. La carpeta firmware/ contiene el código base para el ESP32.

Sección 1.3

1.3 Instalación del frontend

Para instalar dependencias del frontend:

cd /home/abraham/proy_ia_security/frontend
npm install

Para compilar:

npm run build

Para ejecutar en modo producción local:

PORT=3101 npm run start
Para desarrollo local, se puede ejecutar el servidor de desarrollo de Next.js en el puerto definido por el entorno. Durante pruebas, se debe evitar correr dos procesos next dev sobre la misma carpeta y evitar ejecutar npm run build mientras un servidor de desarrollo usa la misma carpeta .next.
Sección 1.4

1.4 Variables de entorno del frontend

En producción, la aplicación debe comunicarse con la API pública:

NEXT_PUBLIC_API_BASE_URL=https://api.afcrseguridad.com

Esta configuración evita problemas de contenido mixto, ya que el frontend público funciona mediante HTTPS y la API pública también debe responder por HTTPS.

Sección 1.5

1.5 Instalación del backend

Para preparar el backend:

cd /home/abraham/proy_ia_security/backend

Se deben instalar las dependencias Python correspondientes al proyecto. Luego, es recomendable verificar la sintaxis del archivo principal: 

python3 -c "import ast, pathlib; ast.parse(pathlib.Path('app_api.py').read_text()); print('app_api.py syntax OK')"

Para ejecutar el backend localmente:

uvicorn app_api:app --host 0.0.0.0 --port 8000
Sección 1.6

1.6 Variables de entorno del backend

El backend requiere variables para configurar servicios externos y parámetros operativos. Entre las más importantes se encuentran: 

PUBLIC_API_URL=https://api.afcrseguridad.com
AI_PROVIDER=openai
OPENAI_MODEL=gpt-4o
OPENAI_TRANSCRIBE_MODEL=gpt-4o-mini-transcribe
OPENAI_TRANSCRIBE_FALLBACK_MODEL=whisper-1
VOICE_PLAN_TTL_SECONDS=300
VOICE_AUDIO_MIN_BYTES=1500
DEVICE_COMMAND_TTL_SECONDS=300

Cuando se usa Supabase, también deben configurarse las variables asociadas a la URL del proyecto y las claves necesarias para que el backend pueda acceder a la base de datos y al almacenamiento privado.

Sección 1.7

1.7 Verificación de funcionamiento

Una vez iniciado el backend, se debe verificar:

curl http://localhost:8000/ping

En producción:

curl https://api.afcrseguridad.com/ping

Si la respuesta indica que el backend está activo, el frontend puede comunicarse con la API.

Sección 1.8

1.8 Consideración para pruebas con ESP32 local

Para pruebas físicas locales, el ESP32 no debe usar localhost. Debe usar una dirección LAN accesible desde el dispositivo, por ejemplo: 

http://192.168.0.5:8000

Antes de cargar el sketch, se debe comprobar desde un celular u otro equipo en la misma red que: 

http://192.168.0.5:8000/ping

Responde correctamente.
