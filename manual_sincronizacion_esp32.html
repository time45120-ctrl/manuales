1. Manual de sincronización ESP32

Este manual describe el proceso para enlazar un ESP32 con la plataforma.

1.1 Generación de token

El usuario ingresa al laboratorio de sincronización y solicita la creación de un token de emparejamiento. El frontend envía esta solicitud al backend mediante el endpoint: 

POST /devices/pairing-token

El backend genera un token temporal y devuelve la información necesaria para configurar el dispositivo.

1.2 Preparación del sketch

El frontend muestra un sketch base para Arduino IDE. El usuario debe copiarlo y editar los siguientes valores: 

const char* WIFI_SSID = "NOMBRE_DE_LA_RED";
const char* WIFI_PASSWORD = "CONTRASENA_DE_LA_RED";
const char* PAIRING_TOKEN = "TOKEN_GENERADO";

También debe verificarse que la URL de la API sea correcta. En producción debe ser: 

https://api.afcrseguridad.com

En laboratorio local debe ser una URL LAN accesible desde el ESP32: 

http://192.168.0.5:8000
No se debe usar localhost en el ESP32 para conectarse al backend local, porque para el ESP32 localhost significa el propio ESP32.

1.3 Carga del firmware

El usuario conecta el ESP32 por USB, abre Arduino IDE, selecciona la placa y el puerto correspondiente, y carga el sketch.

Al iniciar, el ESP32 se conecta al WiFi y solicita reclamar el token.

1.4 Reclamo del dispositivo

El ESP32 llama al endpoint: 

POST /devices/claim

Si el token es válido, el backend registra el dispositivo como enlazado y entrega una clave de API una sola vez. El ESP32 guarda esta credencial localmente para consultar comandos futuros.

1.5 Consulta de comandos

Luego de enlazarse, el ESP32 consulta periódicamente: 

GET /device/commands?device_id={device_id}
Authorization: Bearer <device_api_key>

Si existe un comando pendiente, el dispositivo lo recibe y ejecuta la acción correspondiente, como encender o apagar el LED.

1.6 Confirmación ACK

Después de ejecutar la acción, el ESP32 informa el resultado al backend: 

POST /device/commands/{command_id}/ack

Este ACK permite que el dashboard actualice el estado del comando y muestre al usuario si la acción fue ejecutada correctamente.

1.7 Prueba recomendada

Antes de cargar el firmware en el ESP32, se recomienda validar desde un celular en la misma red: 

http://192.168.0.5:8000/ping

Si esta prueba falla, el ESP32 probablemente tampoco podrá conectarse al backend local.

En ese caso, se debe revisar la IP LAN, las reglas de firewall, el portproxy o la configuración del servidor FastAPI.
