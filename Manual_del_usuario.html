1. Manual del usuario

Este manual describe el uso básico de la aplicación desde la perspectiva de una persona que desea controlar dispositivos mediante comandos de voz.

1.1 Acceso a la aplicación

El usuario ingresa a la aplicación web desde el navegador. En producción, la URL pública del frontend es: 

https://afcrseguridad.com

Desde la pantalla inicial, el usuario puede acceder al entorno de desarrollo, al dashboard de voz o al laboratorio de sincronización, según las opciones habilitadas.

1.2 Verificación del backend

Antes de grabar comandos, el dashboard verifica la disponibilidad del backend mediante el endpoint /ping. Si el backend responde correctamente, el sistema puede continuar con la captura de audio.

Si el backend no responde, el usuario debe revisar la conexión a Internet, la configuración de la URL de API o el estado del servicio backend.

1.3 Grabación de comando de voz

Para enviar una instrucción, el usuario debe permitir el uso del micrófono en el navegador. Luego puede iniciar la grabación y pronunciar un comando, por ejemplo: 

prende el LED

El sistema valida que el audio no esté vacío y que no sea demasiado pequeño. Si el audio parece silencio, el frontend bloquea el envío para evitar interpretaciones incorrectas.

1.4 Revisión de la respuesta

Luego de enviar el audio, el dashboard muestra la respuesta generada por la IA. El usuario debe revisar dos secciones:

  • Respuesta IA para el usuario: explica en lenguaje natural lo que el sistema entendió.
  • Respuesta JSON para el dispositivo: muestra la estructura técnica que será utilizada para automatización.
Esta revisión permite comprobar si la IA interpretó correctamente la orden antes de ejecutar una acción física.

1.5 Confirmación de ejecución

Si el sistema determina que el plan puede ejecutarse, el dashboard habilita la opción de confirmar ejecución. El usuario debe presionar esta opción solo si la acción mostrada coincide con lo que desea realizar.

Al confirmar, el backend coloca el comando en cola para el dispositivo IoT. En el caso del ESP32, el dispositivo recibirá la orden cuando consulte la API.

1.6 Seguimiento del comando

Después de confirmar la ejecución, el usuario puede observar el estado del comando. Los estados posibles son:

  • En cola.
  • Entregado.
  • Ejecutado.
  • Fallido.
  • Expirado.

Si el comando aparece como ejecutado, significa que el dispositivo informó al backend que completó la acción. Si aparece como expirado, significa que el dispositivo no ejecutó el comando dentro del tiempo esperado.

1.7 Interpretación de logs de prueba

El dashboard incluye una sección de logs de prueba. Esta sección ayuda a revisar eventos como permisos de micrófono, tipo MIME de audio, tamaño del archivo, niveles de volumen, respuesta del backend y estado de transcripción.

Estos logs son útiles durante producción temprana porque permiten diagnosticar problemas de micrófono, conexión o interpretación.
